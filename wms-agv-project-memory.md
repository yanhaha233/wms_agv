# WMS-AGV 项目记忆文件

最后更新：2026-07-09

## 项目名称

WMS-AGV 出入库执行仿真系统

英文工作名：WMS AGV Flow Bridge

## 项目定位

这是一个结合你当前工作经验的作品集项目，也是一个后端学习项目。

项目模拟一个 WMS 执行层系统：从模拟 ERP 接收入库单、出库单和基础资料；在 WMS 中人工下发出库单；模拟 AGV 执行出库搬运任务；出库完成后，WMS 将仓库执行账回传给模拟 ERP；入库时现场人员根据到料扫码，叉车将货放到固定点位或电梯，再由 WMS 生成 AGV 入库上架任务，AGV 叉货放到相关点位；系统更新库存、记录库存流水、追踪异常，并提供任务链路和简单统计。

项目不能使用真实公司数据、真实客户名称、真实 AGV 细节、真实数据库结构、真实设备 IP、真实接口报文。所有数据都应该使用模拟数据或脱敏数据。

## 你的工作背景

- 当前工作：WMS 出入库系统，涉及 AGV 对接。
- 上游系统：富勒 ERP 或类似 ERP 系统，提供基础信息、入库单、出库单、物料信息。
- WMS 出库现场流程：
  - ERP 将出库单下发给 WMS。
  - 出库单需要在 WMS 中人工下发。
  - 出库下发后，WMS 生成 AGV 任务。
  - AGV 执行出库搬运并回传状态。
  - AGV 将货送到车间或指定点位后，WMS 判定仓库出库执行完成。
  - WMS 将仓库执行账回传给 ERP，表示仓库已发料或已出库。
- WMS 入库现场流程：
  - ERP 将入库单下发给 WMS。
  - 现场人员根据实际到料扫码。
  - 叉车将货放到固定交接点位，或者通过电梯上楼到指定交接位置。
  - 货放完后，WMS 生成入库任务。
  - AGV 叉货并放到相关库位或点位。
  - AGV 完成后，WMS 记录入库完成、库存落位和库存流水。
- 下游系统：AGV 出入库系统，接收库存、库位、物料、出库单或入库上架任务信息，并自动执行搬运。
- MES 侧：AGV 可能把物料送到 MES 相关位置，但 MES 收到物料之后的具体业务逻辑不在本项目范围内。当前项目将 MES 视为黑盒，不模拟 MES 的生产消耗账、报工账或成本账。

## 项目范围

本项目要做：

- 模拟 ERP 下发入库单、出库单和基础资料。
- WMS 维护物料、库位、库存、入库单、出库单。
- 人工下发出库单。
- 出库前校验库存。
- 生成 AGV 出库搬运任务。
- 模拟 AGV 出库任务执行。
- 出库完成后生成 WMS 仓库执行账。
- 模拟 WMS 将仓库执行账回传 ERP。
- 记录 ERP 回传状态、失败原因和重试次数。
- 入库到货扫码。
- 记录货物放到固定交接点位或电梯点位。
- WMS 根据到货扫码和交接点位生成 AGV 入库上架任务。
- 模拟 AGV 入库上架任务执行。
- 更新库存余额。
- 记录库存流水。
- 记录人工操作日志。
- 记录异常事件。
- 查询出库和入库任务完整时间线。
- 做简单执行统计。
- 提供 2D 实时可视化看板，根据任务事件动态展示货车到货、卸货、扫码、叉车放到交接点、AGV 入库上架、AGV 出库送料等过程。

第一版暂时不做：

- 真实 ERP 对接。
- 真实 AGV 协议对接。
- 真实 MES 业务逻辑。
- MES 生产消耗账或报工账。
- 真实公司数据库连接。
- 大型前端页面。
- Redis、Kafka、Celery、Airflow、机器学习或大数据平台。

## 最终效果

最后项目应该像一个小型 WMS 执行沙盘：

1. ERP 创建或推送入库单、出库单。
2. WMS 展示待处理单据。
3. 操作员人工下发出库单。
4. WMS 校验库存，并生成 AGV 出库任务。
5. 模拟 AGV 按状态执行出库任务。
6. WMS 根据 AGV 回传更新任务状态和库存。
7. 出库完成后，WMS 生成仓库执行账并模拟回传 ERP。
8. 操作员按入库单扫码到货。
9. 系统记录货物已放到固定交接点位或电梯点位。
10. WMS 生成 AGV 入库上架任务。
11. 模拟 AGV 将货物放到相关库位或点位。
12. WMS 更新库存并记录库存流水。
13. 用户可以按单号查看完整执行链路。
14. 用户可以查看异常和简单统计。
15. 用户可以打开实时可视化看板，看到货车、叉车、AGV、交接点、库位、车间点位随着任务状态动态变化。
16. 用户可以按单号回放历史任务，复盘某次入库或出库的完整过程。

## 实时可视化模块

可视化模块目标：

- 让面试官或学习者不用看接口细节，也能一眼看懂 WMS-AGV 出入库流程。
- 用 2D 仓库平面图展示业务状态，而不是一开始做复杂 3D。
- 不是播放固定动画，而是由后端任务事件驱动画面变化。

核心原则：

```text
业务系统负责真实状态
事件中心负责把业务状态翻译成可视化事件
前端地图负责根据事件更新画面
```

推荐架构：

```text
WMS / AGV / ERP 回传状态变化
        ↓
写入业务表和任务事件表
        ↓
生成可视化事件
        ↓
通过 SSE 推送给前端
        ↓
2D 地图更新货车、叉车、AGV、货物、库位状态
```

推荐画面元素：

- 厂区大门。
- 货车。
- 卸货区。
- 扫码点。
- 固定交接点位。
- 电梯点位。
- AGV 等待区。
- AGV 行驶路线。
- 货架库位。
- 出库口。
- 车间收货点。
- 异常提示区域。

入库实时变化：

```text
货车到达卸货区
    ↓
现场人员扫码到货
    ↓
叉车把货放到固定交接点位或电梯点位
    ↓
WMS 生成 AGV 入库上架任务
    ↓
AGV 从等待区出发
    ↓
AGV 到交接点取货
    ↓
AGV 搬运到目标库位
    ↓
WMS 确认库存落位
```

出库实时变化：

```text
ERP 下发出库单
    ↓
WMS 人工下发
    ↓
WMS 分配库位
    ↓
AGV 从库位取货
    ↓
AGV 送到车间或出库点
    ↓
WMS 扣减库存
    ↓
WMS 回传 ERP 仓库执行账
```

可视化事件示例：

```json
{
  "event_type": "agv_task_moving",
  "task_no": "AGV-20260709-001",
  "object_type": "agv",
  "object_code": "AGV-01",
  "from_point": "AGV_WAITING",
  "to_point": "INBOUND_HANDOVER_01",
  "status": "moving",
  "occurred_at": "2026-07-09T18:30:00"
}
```

可视化模式：

- 实时模式：跟随当前正在执行的任务变化，后端产生事件，前端即时更新地图。
- 回放模式：按入库单号、出库单号或 AGV 任务号读取历史事件，按时间顺序重播执行过程。

第一版实时看板建议：

- 后端先生成任务事件。
- 前端先轮询或用 SSE 接收事件。
- AGV 位置可以按起点、终点和状态做简化动画，不需要真实坐标算法。
- 货车、叉车、AGV、货物都可以是简单图标，重点是表达流程、状态和异常。

推荐实现顺序：

1. 先做静态仓库平面图。
2. 再定义可视化事件格式。
3. 再用假事件让货车、叉车、AGV 动起来。
4. 再接后端真实任务事件。
5. 最后加实时模式、历史回放、异常高亮和统计卡片。

第一版可视化不追求真实地图精度，只需要能表达流程和状态。后期如果想升级，可以再考虑更精细的路径、速度、拥堵和多 AGV 同屏。

## ERP 回传边界

WMS 回传 ERP 的内容只表示仓库执行账，例如：

- 哪张出库单已执行完成。
- 实际出库了哪些物料。
- 实际出库数量。
- 批次或托盘信息。
- 源库位。
- 目标点位。
- AGV 任务号。
- 仓库出库完成时间。

MES 回 ERP 的内容属于生产账，例如生产消耗、报工、工单进度、成本核算等。当前项目不模拟这部分，MES 保持黑盒。

本项目需要体现的闭环是：

```text
ERP 下发出库单
    ↓
WMS 执行出库
    ↓
AGV 送达车间或指定点位
    ↓
WMS 生成仓库执行账
    ↓
WMS 回传 ERP
```

## 入库状态建议

入库单状态：

- 待到货
- 部分到货
- 已到货
- 待生成上架任务
- 上架中
- 已完成
- 异常

入库任务状态：

- 待生成
- 待 AGV 接单
- AGV 已接单
- 搬运中
- 已上架
- 失败
- 取消

库存状态：

- 预期库存：ERP 入库单上的应收数量。
- 暂存库存：现场扫码后，货物在固定交接点或电梯点位。
- 在途库存：AGV 已接单并正在搬运。
- 可用库存：AGV 上架完成后，进入正式库位。

目标：

- 模拟 ERP 下发入库单和出库单。

任务：

- 模拟 ERP 创建单据。
- 入库单。
- 出库单。
- 单据明细。

- 重复单据幂等处理。

验收标准：

- 同一张单据重复推送，不会重复生成业务数据。

### 阶段 4：出库下发与 AGV 执行

目标：

- 跑通一条完整出库执行链路。

任务：

- 操作员人工下发出库单。
- WMS 校验库存。
- WMS 分配库位库存。
- WMS 创建 AGV 出库任务。
- 模拟 AGV 回传状态。
- WMS 完成出库并扣减库存。
- WMS 生成仓库执行账。
- WMS 模拟回传 ERP 出库执行结果。

验收标准：

- 示例：库存 100，出库 20，AGV 完成后库存变成 80，并记录库存流水；同时生成一条 ERP 回传记录，表示仓库已完成出库执行。

### 阶段 5：入库到货扫码与 AGV 上架

目标：

- 跑通入库从到货扫码到 AGV 上架完成的流程。

任务：

- 操作员根据入库单扫码物料、批次、数量。
- 记录货物放置的固定交接点位或电梯点位。
- 校验入库单。
- 支持部分到货。
- 防止超收到货。
- WMS 生成 AGV 入库上架任务。
- 模拟 AGV 接单、搬运、上架完成。
- 上架完成后增加正式库位库存。
- 记录库存流水。

验收标准：

- 示例：应收 100，第一次到货扫码 60 放到交接点，WMS 生成 AGV 上架任务；AGV 上架完成后，目标库位库存增加 60，并记录库存流水。

### 阶段 6：异常中心与任务追踪

目标：

- 让系统更接近真实现场。

任务：

- 记录库存不足。
- 记录未知物料或未知库位。
- 记录未知交接点位。
- 记录重复单据。
- 记录 AGV 超时或失败。
- 记录 AGV 状态流转错误。
- 记录 ERP 回传失败。
- 记录 ERP 回传重复或幂等问题。
- 记录入库超收。
- 记录扫码后未生成上架任务。
- 记录 AGV 到达错误点位。
- 建立单据执行时间线。

验收标准：

- 输入出库单号或入库单号，可以看到从 ERP 来单到完成或异常的完整时间线。
