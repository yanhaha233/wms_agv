# 项目故事与边界

## 一句话说明

WMS-AGV 出入库执行仿真系统用于模拟一个仓库执行层闭环：ERP 下发出入库需求，WMS 管理单据、库存和任务，AGV 仿真执行搬运，WMS 记录执行结果并回传 ERP。

## 为什么做这个项目

这个项目服务两个目标：

- 业务目标：把 WMS 出入库、AGV 任务、库存流水、异常追踪和 ERP 回传串成一条能讲清楚的业务链路。
- 学习目标：用一个贴近真实工作的场景练习 Python、FastAPI、PostgreSQL、SQLAlchemy、Alembic、测试、Docker 和后端项目拆分。

它不是企业级 WMS，也不是 AGV 调度系统。它是一个可演示、可学习、可逐步扩展的后端作品集项目。

## 系统边界

系统内负责：

- 接收或模拟 ERP 基础资料、入库单和出库单。
- 维护物料、库位、交接点位、库存余额和库存流水。
- 支持 WMS 操作员人工下发出库单。
- 在出库前校验库存并生成 AGV 出库任务。
- 模拟 AGV 接单、搬运、完成和失败状态。
- 在 AGV 完成后更新库存并生成仓库执行账。
- 模拟 WMS 回传 ERP 仓库执行结果。
- 支持入库到货扫码、交接点暂存和 AGV 上架。
- 记录操作日志、接口日志、异常事件和任务时间线。

系统外暂不负责：

- 真实 ERP 系统接口。
- 真实 AGV 通讯协议。
- 真实 MES 生产消耗、报工、成本或工单进度。
- 真实客户资料、真实设备 IP、真实点位图、真实数据库结构。
- 复杂调度算法、多车路径规划和交通管制。

## MES 边界

MES 在本项目中保持黑盒。AGV 可能把物料送到车间或 MES 相关点位，但 MES 收到物料之后的生产消耗账、报工账和成本账不进入第一版范围。

本项目重点闭环是：

```text
ERP 下发单据
WMS 执行仓库业务
AGV 仿真搬运
WMS 更新库存和仓库执行账
WMS 回传 ERP
```

## 数据安全约定

项目只能使用模拟数据或脱敏数据。仓库编码、物料编码、客户名称、接口字段、设备编号和设备地址都不能来自真实生产环境。

推荐使用这样的模拟编码：

- 物料：MAT-001、MAT-002。
- 库位：A01-01-01、A01-01-02。
- 交接点：HANDOVER-IN-01、ELEVATOR-01。
- AGV：AGV-01、AGV-02。
- 单据：IN-20260710-001、OUT-20260710-001。

## 阶段 0 交付物

阶段 0 只做定位与设计，不写业务代码。交付物包括：

- README 项目说明。
- 业务流程图。
- 角色与职责说明。
- 核心对象与状态说明。
- 第一版边界和后续阶段说明。

完成后，另一个人应该能只看仓库文档就理解项目要解决什么问题。
