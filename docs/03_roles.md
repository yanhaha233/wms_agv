# 角色与职责

## 模拟 ERP

职责：

- 下发物料、库位等基础资料。
- 下发入库单和出库单。
- 接收 WMS 回传的仓库执行结果。

不负责：

- 直接修改 WMS 库存。
- 直接控制 AGV。
- 处理 MES 生产消耗账。

## WMS 操作员

职责：

- 查看 ERP 下发的待处理单据。
- 人工下发出库单。
- 处理异常，例如库存不足、未知物料、重复单据。
- 查看任务时间线和执行统计。

关键动作：

- 下发出库。
- 查询库存。
- 查看异常。
- 重新触发或标记失败任务。

## 现场操作员

职责：

- 根据入库单扫码确认到货。
- 录入物料、批次、数量和到货时间。
- 发现实物与单据不一致时上报异常。

关键动作：

- 到货扫码。
- 确认部分到货。
- 标记超收或物料不匹配。

## 叉车人员

职责：

- 把到货物料放到固定交接点位或电梯点位。
- 确认货物已具备 AGV 接驳条件。

关键动作：

- 确认交接点位。
- 确认货物已放置。

## AGV 仿真系统

职责：

- 接收 WMS 生成的入库上架任务或出库搬运任务。
- 模拟接单、搬运、完成和失败状态。
- 将任务状态回传给 WMS。

不负责：

- 真实路径规划。
- 真实交通管制。
- 真实设备通讯。

## 仓库管理员

职责：

- 查看库存余额、库存流水和仓库执行账。
- 查看出入库任务完整链路。
- 查看异常统计和执行统计。
- 用项目演示业务闭环。

关键动作：

- 按单号查询时间线。
- 按日期查看入库、出库和异常统计。
- 复盘 AGV 任务执行过程。

## 系统边界角色

MES：

- 在第一版中视为黑盒。
- 可以作为出库目标点位存在。
- 不模拟生产消耗、报工、工单进度或成本。

真实 ERP 和真实 AGV：

- 第一版不接入。
- 仅保留模拟接口和模拟状态，方便后续替换真实适配层。
